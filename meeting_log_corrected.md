## Основные продукты для пользователей

В Яндекс.Бизнесе существует два основных типа размещения:
- **"Подписка"** — новый дизайн интерфейса
- **"Приоритетное размещение"** — старый интерфейс

### 1. Приоритетное размещение

**Ключевые ограничения:**
- Доступно только для сетей (несколько филиалов)
- Для одиночных организаций доступен только "брендированный приоритет"
- Клиент не может самостоятельно создать "приоритетное саморазмещение" — только через менеджера

**Важно:** Если менеджер выбирает "приоритетное размещение" для одиночной организации, фактически создается подписка с включенными Картами, а не настоящее "приоритетное размещение".

**Андрей:** Если я ресторан в Ташкенте, я не могу купить приоритет, а только брендированный приоритет?
**Елена:** Либо брендированный приоритет, либо подписку с включенными только картами.

**Причины сохранения приоритетного размещения**
- Сохранено только для сетей из-за особенностей ценообразования
- Для сетей с большим количеством филиалов (20-30) подписка становится очень дорогой
- В приоритетном размещении действуют скидки за количество филиалов
- Переписывание биллинга было слишком дорогим

### 2. Подписка

**Создание подписки через менеджера:**
- Только менеджер может создать подписку с включенными только картами
- Клиент может самостоятельно добавить дополнительные площадки (например, Яндекс.Поиск)
- **Важное ограничение:** Убрать добавленные площадки клиент самостоятельно не может — только через менеджера

**Самостоятельное создание подписки:**
- Клиент создает подписку на все места размещения
- Отключить отдельные площадки можно только через менеджера

**Специальная ссылка для акций:**
- Существует специальная ссылка для создания подписки только с картами
- Создана для акции "хорошее место" (запускается раз в год)
- Ссылка всегда активна и не привязана к конкретной акции

**Вадим:** Если мы в международном сегменте захотим давать клиенту возможность создавать подписку с только картами, мы можем ему подсовывать эту ссылку?
**Елена:** Способ законный. Возможно, для международного сегмента потребуется доработка, если функция реализована только для российских компаний.

## Визуальное отображение на картах

**Единообразие отображения:**
При оформлении любого типа размещения (подписка для одиночных организаций или приоритетное размещение для сетей) на карте отображаются одинаковые варианты:
- Приоритет
- Брендированный приоритет  
- Брендированный приоритет с POI

**Различие:** Только в способах заведения через интерфейс.

## Ценообразование и факторы стоимости

**Факторы, влияющие на стоимость**
1. **Рубрика** организации
2. **Регион** размещения
3. **История организации** (точно есть в подписке, в приоритете — неизвестно)
4. **Скидка за количество филиалов** (для сетей)

**Аукционная модель**
Все размещение теперь работает через аукцион Директа.

**Елена:** "Сейчас у нас всё размещение на картах проходит через Директ, через алгоритмы Директа"

### Эволюция системы ценообразования

**Старая модель (без аукциона):**
- Фиксированная стоимость независимо от суммы (10к или 15к — одинаковый эффект)
- Крупные клиенты получали большие скидки и "откручивались" на суммы, превышающие их платежи

**Новая модель (с аукционом):**
- Деньги напрямую влияют на результат через аукцион
- Скидки за объем и количество филиалов снижают минимальную цену входа
- Клиенты "откручиваются" за те деньги, которые заплатили

**Вадим:** Стал ли аукцион хуже для СМБ, чем прежнее решение?
**Елена:** Аукцион направлен на оптимизацию и более эффективное распределение денег клиентов. Для пользователя визуально мало что изменилось, но крупные клиенты могли просесть в конверсии. На самом деле их конверсия вернулась к справедливым числам.

## Техническая реализация биллинга в Я.Бизнесе

Как решается, сколько в итоге списывается в аукционе?

**Механизм работы:**
1. В Директе создается рекламная кампания
2. Устанавливается еженедельный лимит списания
3. Директ разыгрывает баннеры по своим алгоритмам
4. Аукцион рассчитывает списание на заданную сумму за недельный период

**Процесс выбора тарифа:**
1. Система рассчитывает три варианта бюджета: минимальный, оптимальный, максимальный
2. Пользователь может выбрать любой бюджет (не меньше минимального)
3. Выбирается срок размещения: 3, 6, 9 месяцев
4. После оплаты общий платеж делится на выбранный период
5. Ежедневно списывается одинаковая сумма

**Технические отличия!** 
"С точки зрения разработки Биллинг Подписки и Биллинг Приоритета — это совершенно разные вещи. При обращении к разработчикам нужно всегда конкретизировать, о каком сервисе идет речь."
